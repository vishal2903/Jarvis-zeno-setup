The Problem

Mentors spend hours manually creating lesson plans, slides, and practice sets for each session. Repetitive work steals time from actual teaching.

How Zeno Works

1

Voice Conversation (HITL)

2

RAG-Powered Context

Searches 50+ lectures via FAISS semantic search to surface related prior content.

Proprietary two-phase engine: LLM parses intent into JSON spec, then deterministic renderer produces branded PPTX/DOCX. 10x cheaper than raw LLM generation.

Lesson plan + slides + practice sets uploaded automatically to Google Drive via OAuth2. Zero manual steps.

Custom Skill Infrastructure

Why We Built This

Anthropic's Skills API with raw LLM generation was costing 10x more per deck and producing inconsistent, off-brand output. We replaced the expensive free-form generation with a deterministic rendering engine — built entirely from scratch.

Low-Level Design

Under the Hood

Slide Deck Generator

Skills/slide-deck/generator/

parser.py — LLM extracts slide JSON spec from lesson plan. Runs Claude API call with typed schema.

schema.py — PresentationSpec + SlideSpec Pydantic models. Validates LLM output.

renderer.py — SlideRenderer dispatches each slide to typed render function via python-pptx.

slide_types.py — 9 typed slide renderers: title, question, comparison, reveal, process, definition, architecture, best-practice, takeaway.

styles.py — Exact RGB colors, Space Grotesk font, coral (#F96846) palette enforced.

Output: 13-20 slide PPTX, 10"x7.5" 16:9, pixel-perfect branding. Uploaded to Google Drive/Slides/

Practice Set Generator

Skills/practice-set-extracted/generator/

ps_parser.py — LLM extracts 3-level exercise spec (Beginner/Intermediate/Advanced) with web search for real-world scenarios.

ps_schema.py — PracticeSetSpec + DifficultyLevel Pydantic models.

ps_renderer.py — PracticeSetRenderer builds DOCX via python-docx with JetBrains Mono for code blocks.

ps_styles.py — Colors, Fonts, FontSizes constants. Consistent brand across every document.

ps_utils.py — JetBrains Mono auto-download. Table styles. Code block formatting.

Output: DOCX with 3 difficulty tiers, real-world problem statements. Uploaded to Google Drive/Practice/

Voice Agent Flow

End-to-End Session Flow

HITL Discovery Phase

1

"jarvis" wake word triggers session

2

Mentor speaks topic (e.g., "multi-agent systems")

3

Agent runs search_curriculum → FAISS top-5

4

Runs get_full_lecture → 4000-char context

5

7 discovery questions (Socratic, voice, one-at-a-time)

6

generate_lesson_plan worker (400s, separate API call)

Content Generation Phase

Lesson Plan Worker

Extracts HITL chat history → Claude Haiku (400s timeout) → 1500-word Markdown → Drive/LessonPlans/

Slide Generation (voice: "make slides")

skills_handler.generate_slides() → upload .md → Anthropic Skills container → Custom Skill Infrastructure → PPTX → Drive/Slides/

Practice Set (voice: "make practice")

skills_handler.generate_practice_set() → same pipeline → DOCX → Drive/Practice/

End State

Agent speaks Drive links. User says "good night" → session closes. Zero manual steps taken.
